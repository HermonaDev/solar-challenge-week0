# Development Guidelines

## Branch Strategy
- `main` - Production-ready code only
- `feature/description` - New features and analysis
- `fix/issue` - Bug fixes and improvements

## Commit Standards
- `feat: add solar data cleaning pipeline`
- `fix: handle negative irradiance values` 
- `docs: update project documentation`
- `analysis: add country EDA`

## Code Quality
- Use docstrings for all functions
- Reusable code in `scripts/` directory
- Analysis code in `notebooks/` directory
- Tests in `tests/` directory
